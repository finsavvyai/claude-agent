```markdown
# How I Built Semantic Code Search with Claude AI and Vector Embeddings

## The Problem

Traditional code search (like GitHub search or grep) only finds exact keyword matches.

But developers don't think in keywords. They think in concepts:

- "Where is the authentication logic?" (not searching for "auth" string)
- "How do we handle payments?" (not searching for "stripe")
- "What APIs does the mobile app call?" (semantic understanding needed)

I built Luna Agents to solve this with semantic code search.

## Architecture Overview

```
User Query ‚Üí Embedding Model ‚Üí Vector Search ‚Üí Claude AI ‚Üí Formatted Answer
```

## Step 1: Code Ingestion

When you connect a GitHub repo, we:

1. Clone the repo
2. Parse all files (ignore node_modules, .git, etc.)
3. Split into chunks (max 1000 tokens per chunk)
4. Generate embeddings using `text-embedding-ada-002`
5. Store in Pinecone vector DB

**Code snippet:**
```javascript
async function ingestRepository(repoUrl) {
  const files = await cloneAndParse(repoUrl);

  for (const file of files) {
    const chunks = splitIntoChunks(file.content, 1000);

    for (const chunk of chunks) {
      const embedding = await openai.embeddings.create({
        model: "text-embedding-ada-002",
        input: chunk.text
      });

      await pinecone.upsert({
        id: `${file.path}:${chunk.id}`,
        values: embedding.data[0].embedding,
        metadata: {
          filepath: file.path,
          language: file.language,
          chunk_text: chunk.text,
          line_start: chunk.lineStart,
          line_end: chunk.lineEnd
        }
      });
    }
  }
}
```

## Step 2: Semantic Search

When user asks a question:

1. Convert query to embedding
2. Vector search in Pinecone (cosine similarity)
3. Return top 10 most relevant code chunks

**Code snippet:**
```javascript
async function semanticSearch(query, repoId) {
  // Convert query to embedding
  const queryEmbedding = await openai.embeddings.create({
    model: "text-embedding-ada-002",
    input: query
  });

  // Search Pinecone
  const results = await pinecone.query({
    vector: queryEmbedding.data[0].embedding,
    topK: 10,
    filter: { repo_id: repoId },
    includeMetadata: true
  });

  return results.matches.map(match => ({
    filepath: match.metadata.filepath,
    code: match.metadata.chunk_text,
    lines: `${match.metadata.line_start}-${match.metadata.line_end}`,
    similarity: match.score
  }));
}
```

## Step 3: Claude AI Synthesis

Take the 10 code chunks and send to Claude with:

```javascript
const prompt = `
You are a code search assistant.

User Question: ${query}

Relevant Code Chunks:
${chunks.map(c => `
File: ${c.filepath} (lines ${c.lines})
\`\`\`${c.language}
${c.code}
\`\`\`
`).join('\n')}

Provide a concise answer with:
1. Direct answer to the question
2. File paths where relevant code lives
3. Code snippets (with line numbers)
4. Brief explanation of how it works
`;

const answer = await anthropic.messages.create({
  model: "claude-sonnet-4",
  messages: [{ role: "user", content: prompt }]
});
```

- Query: "authentication logic"
- Results: 847 files with word "auth"
- Time to find answer: 2.5 hours

**After (Luna semantic search):**

## Performance Optimizations

1. **Caching**: Cache embeddings for 7 days
2. **Batch processing**: Generate embeddings in batches of 100
3. **Lazy loading**: Only embed files when repo is queried
4. **Incremental updates**: Only re-embed changed files on git push
